# Methodology

## Introduction

The objective of this project is to automate the analysis of Linux authentication logs using Bash scripting. The project follows a structured workflow similar to a Security Operations Center (SOC), where authentication events are collected, analyzed, reported, and archived automatically.

Rather than manually inspecting thousands of log entries, the developed scripts automate repetitive security monitoring tasks, allowing administrators to identify suspicious login attempts and review system activity more efficiently.

---

# Methodology Overview

The project consists of six major stages:

1. Environment Setup
2. Log Collection
3. Security Event Generation
4. Log Analysis
5. Report Generation
6. Automation

---

# Phase 1 – Environment Setup

An Ubuntu virtual machine was configured as the primary monitoring system. Required packages including OpenSSH Server, Git, Tree, and Net-tools were installed.

A Kali Linux virtual machine was configured on the same virtual network to simulate authentication events against the Ubuntu server.

The Ubuntu system maintains several log files within the `/var/log` directory, with authentication events primarily recorded in `auth.log`.

---

# Phase 2 – Log Collection

Linux continuously records authentication events generated by user logins, SSH sessions, and privilege escalation.

The following logs were examined:

| Log File | Purpose |
|----------|---------|
| auth.log | Authentication events |
| syslog | System messages |
| kern.log | Kernel events |
| dpkg.log | Package installation history |
| journal | Systemd logs |

The authentication log (`auth.log`) served as the primary data source throughout this project.

---

# Phase 3 – Security Event Generation

To generate realistic security data, authentication events were simulated from the Kali Linux virtual machine.

The following scenarios were performed:

- Multiple failed SSH login attempts
- Successful SSH authentication
- Invalid username login attempts
- Sudo command execution
- User session creation and termination

These activities generated entries inside `/var/log/auth.log`, allowing the Bash scripts to process realistic security events.

---

# Phase 4 – Log Analysis

Several Bash scripts were developed to automate log analysis.

The scripts performed the following operations:

- Search for failed login attempts
- Detect successful SSH logins
- Monitor sudo activity
- Extract usernames
- Extract source IP addresses
- Count authentication events
- Detect repeated failed logins
- Identify potentially suspicious IP addresses

Linux utilities including `grep`, `awk`, `sort`, `uniq`, `wc`, and `tar` were used throughout the project.

---

# Phase 5 – Report Generation

The collected information was summarized into human-readable reports.

Each report includes:

- Date and time
- Hostname
- Current user
- Kernel version
- Number of failed logins
- Number of successful logins
- Number of sudo events
- Current logged-in users
- Top failed login IP addresses

These reports provide administrators with a quick overview of recent authentication activity.

---

# Phase 6 – Automation

Automation was implemented using Cron.

Cron schedules execute the reporting and backup scripts at predefined intervals without requiring manual intervention.

Automated tasks include:

- Generate security reports every six hours
- Create daily compressed log backups

Automation reduces administrative effort while ensuring regular monitoring of authentication logs.

---

# Testing Methodology

The project was tested using both normal and malicious authentication scenarios.

Testing included:

- Valid SSH login
- Invalid password attempts
- Invalid username attempts
- Multiple failed logins
- Sudo command execution
- Log backup verification
- Cron schedule verification

Each script was executed individually before integrating it into the automated workflow.

---

# Results

The developed solution successfully:

- Detected authentication failures
- Identified successful SSH logins
- Extracted login IP addresses
- Detected repeated failed login attempts
- Generated automated daily security reports
- Archived important system logs
- Automated routine monitoring tasks

The project demonstrates how Bash scripting can significantly reduce manual log analysis while improving the efficiency of Linux security monitoring.

---

# Limitations

The current implementation has several limitations:

- Supports Ubuntu-style authentication logs.
- Does not provide real-time monitoring.
- Does not send alerts or notifications.
- Does not integrate with external SIEM platforms.
- Uses static threshold values for suspicious login detection.
